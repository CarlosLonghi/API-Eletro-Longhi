## Template de Feature Spec

Use este template para documentar nova feature:

```markdown
# Feature Spec: [Nome da Feature]

**Owner:** [Seu nome]  
**Status:** Draft / In Review / Approved / Implemented  
**Data:** YYYY-MM-DD

## Descrição

[Breve descrição do que é a feature]

## Caso de Uso

[Quem usa? Por quê? Problema que resolve?]

## Requisitos Funcionais

- [ ] Requisito 1
- [ ] Requisito 2

## Requisitos Técnicos

- Endpoints necessários
- DTOs
- Validações
- Integrações

## Casos de Uso (User Stories)

### Como [usuário], eu quero [ação], para que [benefício]
- Cenário 1: ...
- Cenário 2: ...

## Critério de Aceitação

- [ ] Critério 1
- [ ] Critério 2

## Notas para QA

[O que testar? Casos edge?]

## Notas para Operação

[Qualquer step especial de deploy ou monitoração?]
```

---

## Boas Práticas

- Mantenha specs atualizadas quando feature muda
- Refira exec-plans relativas em docs/exec-plans/
- Use linguagem clara (não técnica demais)
- Inclua exemplos de requests/responses
