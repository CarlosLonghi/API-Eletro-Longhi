# design-docs — Índice

Coloque aqui os documentos de design detalhados, decisões arquiteturais e análises aprofundadas.

---

## Arquivos Relacionados

- `core-beliefs.md` — Princípios e crenças fundamentais que guiam a arquitetura

---

## Como Adicionar Novo Design Doc

1. Crie um arquivo markdown (ex: `async-processing-strategy.md`)
2. Comece com título e contexto do problema
3. Liste opções consideradas, trade-offs
4. Documente decisão final e impacto
5. Atualize este índice com link

---

## Exemplo de Estrutura

```markdown
# Design Document: [Título]

**Data:** YYYY-MM-DD  
**Owner:** [Seu nome]  
**Status:** Draft / Approved / Implemented

## Problema

[Descrever problema]

## Opções Consideradas

- Opção 1: ...
- Opção 2: ...

## Decisão

[Decisão tomada]

## Impacto

- Afeta: [que componentes]
- Trade-offs: [o que ganhamos/perdemos]

## Próximos Passos

[Tarefas de follow-up]
```
